This roadmap outlines the strategic focus areas for the UI Guild to evolve frontend architecture, improve developer experience, and scale the Nebula Design System across platforms.
The initiatives move through a lifecycle of exploration → proof of concept → standards → adoption, ensuring experimentation leads to actionable outcomes for product teams.

1. Micro Frontend + BFF Strategy
Objective

Define a scalable frontend architecture that enables teams to deliver independently while maintaining consistency and performance.

Focus Areas

Micro frontend architecture patterns

Backend-for-Frontend (BFF) patterns for domain-based APIs

Communication and integration between MFEs

Deployment, versioning, and ownership models

Expected Outcomes

Reference architecture for Micro Frontends

BFF integration patterns for frontend teams

Architecture decision records and implementation guidelines

Example reference implementations

This initiative ensures teams can scale frontend development across domains without introducing architectural fragmentation.

2. State Management Strategy
Objective

Establish a clear and consistent state management approach for Angular applications.

Focus Areas

Evaluation of modern Angular state management approaches

Comparison between signals-based state and traditional global state management

State boundaries in distributed frontend architectures

Patterns for local versus global state

Expected Outcomes

Recommended state management strategy

Guidelines for structuring application state

Example implementations demonstrating recommended patterns

Documentation for teams adopting the approach

The goal is to reduce architectural inconsistency while improving maintainability and developer productivity.

3. Mobile Strategy for Nebula Design System
Objective

Define how the Nebula Design System supports mobile experiences while maintaining design consistency and enabling efficient development across platforms.

As products increasingly deliver mobile experiences, Nebula must evolve to support:

responsive and mobile-first UI patterns

touch-friendly components

cross-platform design consistency

mobile application architectures

Without a clear strategy, teams may create independent mobile solutions, resulting in duplicated work and inconsistent user experiences.

Strategic Approaches to Evaluate
Hybrid Mobile (Angular + Capacitor)

One approach is using Angular applications wrapped as native mobile apps with Capacitor and Angular.

This approach allows:

reuse of existing Angular components

shared codebases between web and mobile

access to native device capabilities such as camera, notifications, and biometrics

Key evaluation areas include:

performance on mobile devices

responsiveness of Nebula components

integration with native APIs

developer workflow for mobile development

This model prioritizes code reuse and faster delivery.

Native Mobile (React Native)

Another option is building a native mobile component implementation using React Native.

In this model:

Nebula has separate web and mobile implementations

design tokens and visual standards remain shared

This approach prioritizes:

native performance

mobile interaction patterns

deeper platform integration

However, it introduces additional complexity due to maintaining multiple component implementations.

Design System Considerations

Regardless of the architecture chosen, Nebula must support mobile by introducing:

responsive component design

mobile interaction patterns

shared design tokens

cross-platform design consistency

Some components may require mobile-specific behaviors, such as adapted navigation patterns, modal interactions, or layout transformations.

Role of Component Development Tools

Tools such as Storybook can support mobile readiness by enabling:

mobile viewport previews

component documentation and usage examples

visual regression testing

validation of responsive behavior

This helps ensure components behave consistently across devices before being adopted by product teams.

AI Opportunities

Emerging AI tools may help reduce the effort of supporting multiple platforms.

Tools such as:

Claude

GitHub Copilot

can be explored to support:

generating mobile component implementations

converting Angular components into mobile equivalents

generating templates and component scaffolding

improving documentation and developer workflows

AI-assisted tooling may significantly accelerate the evolution of the design system across platforms.

Expected Outcomes

clear mobile architecture recommendation

mobile-ready Nebula component guidelines

reference implementations demonstrating the approach

documented workflows for building mobile experiences

This initiative ensures Nebula evolves into a cross-platform design system capable of supporting both web and mobile products.

4. Accessibility
Objective

Improve accessibility across UI applications and embed accessibility practices into the development lifecycle.

Focus Areas

accessibility standards for Nebula components

automated accessibility testing

screen reader compatibility

accessibility awareness across development teams

Tools and Exploration

Introduce visual and accessibility testing using tools such as Chromatic and explore AI-assisted validation techniques.

Expected Outcomes

accessibility guidelines for UI teams

automated accessibility checks in CI pipelines

improved accessibility awareness through guild workshops

improved baseline accessibility scores across applications

This initiative ensures accessibility becomes a built-in part of the UI development workflow rather than a late-stage validation step.

5. AEM Cloud Strategy
Objective

Define how frontend applications integrate with Adobe Experience Manager Cloud.

Focus Areas

headless versus hybrid AEM architectures

integration with micro frontend environments

content modeling strategies

frontend rendering approaches

Expected Outcomes

recommended frontend integration architecture

reusable integration patterns

example implementations demonstrating best practices

This initiative ensures that UI teams can efficiently integrate content management capabilities into frontend applications.

6. AI Exploration for UI Development
Objective

Explore how artificial intelligence can improve the UI development workflow, design system operations, and developer productivity.

Focus Areas

Design workflow

Exploration of AI capabilities in tools such as Figma to assist with design generation, layout suggestions, and design token creation.

Product management workflow

Evaluation of AI capabilities in tools such as Jira to support task creation, documentation, and workflow automation.

Developer experience

Investigation of AI-driven tooling to support:

automated component generation

template scaffolding

design system documentation

developer assistants for UI development

Expected Outcomes

identification of high-impact AI opportunities

experimentation with AI-assisted component generation

improved developer productivity through AI tooling

This initiative positions the UI Guild to leverage AI as a force multiplier for design system development and frontend engineering workflows.
